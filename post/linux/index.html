命令行

  • 文件

    • mkdir: 新建文件夹
    • touch： 新建文件
    • gedit：打开文件
    • 删除目录： rm -rf dirname
    • 删除文件：rm filename
    • 清空目录: rm -rf *
    • 复制粘贴： cp file dir
    • 剪切粘贴： mv file dir
    • 复制目录： cp -r dir_src dir_tgt
    • 剪切目录： mv dir_src dir_tgt
    • 目录重命名：mv name1 name2
    • 显示第2层文件结构： tree -L 2
    • 以写的形式打开文件：open(“path”, ‘w’)

  • 查询

    • 查询内存：free -m （以M为单位显示）
    • 查询显存和GPU：nvidia-smi (主要看memory-usage：显存使用；GPU-Util： GPU利用率；Processes: 各进程占用显存情况)
    • 查询当前目录文件夹大小：du -h(human-readable) -d1(深度1层) -c(count total) -s(summarize)
    • 查询当前目录文件大小: ls -lh
    • 查询磁盘： df -a(all) -h(human-readable) -i(inode) -T(type)
    • 查询当前目录大小：ls -lh
    • 动态查询GPU： nvidia-smi dmon
    • 动态查询CPU： top
    • top -p pid 可以查询pid信息
    • 查询在线用户：who/last
    • 查询指定端口使用情况： netstat -tunlp | grep 8080
    • 查找文件：find dir -name fine_name 如： find . -name “log”
    • watch -n 0.1 nvidia-smi
    • 查询磁盘使用情况： baobab
    • 查询硬件温度：安装psensor

  • 文件传递（在本地命令行）

    • 上传本地文件到服务器： scp local_file username@servername:remote_dir
    • 下载服务器文件到本地： scp username@servername:remote_file local_dir
    • 上传本地目录到服务器： scp -r local_dir username@servername:remote_dir
    • 下载服务器目录到本地： scp -r username@servername:remote_dir local_dir

  • 安装

    • apt
      • 【离线安装】apt-get install 的deb安装包一般存储在：/var/cache/apt/achives
      • sudo apt-get update 读取源地址中最新的软件版本列表，并保存在电脑中，apt-get install通过该列表下载
      • sudo apt-get upgrade 将本地已安装的软件与最新的软件版本列表对比，如果本地版本低于最新版本，会提示更新
      • 自动解决安装包的依赖关系，自动搜索安装其他依赖文件
    • dpkg
      • 不会自动搜索该软件包的依赖项。若安装时提示缺少依赖，执行依赖修复(sudo apt-get install -f) 再用dpkg安装deb软件包
      • 离线安装：dpkg -i name.deb
    • snap
    • bash setup.sh. 或者 ./setup.sh

  • 添加环境变量

    • 查找文件夹（如anaconda下的bin文件夹）： whereis anaconda
    • 写入环境变量到/etc/paths ：如（追加） export PATH=/opt/local/bin:/opt/local/sbin:$PATH 又如： （添加）export CUDA_HOME=“/usr/local/cuda”
    • 查找环境变量:
      • 查看单个环境变量echo $PATH 又如：echo $CUDA_HOME
      • 查看所有环境变量：export 或者 env

      • ssh user@ip 连接远程服务器
      • ssh user@ip -L local_port:localhost:remote_port 连接远程服务器，并从本地端口访问远程服务器端口（Tensorboard, Jupyter Notebook）
      • ssh xieliang@10.13.22.251 -L 6006:localhost:remote_port 打开tensorboard
      • ssh xieliang@10.13.22.251 -L 8008:localhost:remote_port 打开jupyter notebook

      • local_port需要使用可用的端口(可能被占用)，一般tensorboard默认6006， jupyter notebook默认8008
